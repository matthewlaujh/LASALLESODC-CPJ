// 19.10.21 - Arduino Exploration: Conductive Paint with WIFI.

The next part was trying it out with the Arduino Uno Wifi, now both the Uno WIFI and Uno are supposed to be sort of like the same board just with added functionality but somehow, there might be some minor differences which affected how the capacitive sense library worked. I tried running the same code that worked on the Uno board, but it repeatedly gave me this "-2" sensor value (instagram video below where I uploaded the same code and used the circuits with the two different boards). So according to the Capacitive Sensor Library Documentation there are two possible problems:

capacitiveSensor and capacitiveSensorRaw will return -2 if the methods timeout. This is caused by the count exceeding the value of CS_Timeout_Millis, which is set at a default value of 2000 milliseconds (2 seconds). This is most often caused by a missing resistor or the resistor in the wrong pin. It could also be caused by a sensor that is grounded or connected to +5 V.

So I did try messing around with the Timeout settings but that didn't make any difference. I also tried messing around with the code a bunch as well as rewiring the circuits to double check, but once again no difference. One way that I found that it worked was actually unplugging one the non-input wire, so just having like a broken circuit, but that wasn't really working either? It was getting values, but like large numbers (instagram video below), so that wasn't very practical either. So I decided to ask Andreas for some help, but ok funny story when he tried it, it actually worked for a moment, then when I tried it again when he left, it stopped working. I really have no clue why and how but I need something for class so I had to find another thing to work on, so this is now on pause.

Upon further digging, I also found that there hasn't been a new release for the library since 2016 (with the exception of a small update in 2020, providing support for another board that I use, the Arduino Nano 33 BLE SENSE, might try something with that board later on), and the Arduino Uno WIFI Rev 2 board was released in 2018 so what I was suspecting is that the library did not support the board which was why it didn't work. I also didn't see it listed as a supported board and I saw on the issues page of the library's github there were other people facing the issue of "-2" values when using other boards that weren't supported. So this is highly likely the case and I guess I've sort of hit a dead end for now unless I can find a way to resolve things.

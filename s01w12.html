// 01.11.21 - Arduino + Machine Learning Exploration: TinyML using Tiny Motion Trainer and Arduino with BLE LED output.

With the wired connection working, next was to use the BLE (Bluetooth Low Energy) feature of the Arduino Nano 33 BLE Sense to communicate between the two devices so that (hopefully) it could be a wireless interface on your finger, wouldn't make much sense to have a long cable to connect the devices. Ideally I hoped for the devices to have WIFI, but I just have to work with what I have.

Although there's a possible solution I could explore later on where I pair an Arduino Nano 33 BLE Sense (Finger Interface) to an Arduino Nano 33 IOT (Output Device). There's a chance that the Nano IOT could run both BLE and WIFI at the same time (although it requires some advanced hacking according to the Arduino Documentation, as the feature is quite new and still being developed) but theoretically that could work. But I do not want to mess too much with the firmware on the boards now with submission coming up, so this could be explored during the holidays.

First up is getting the two devices to connect over BLE. I followed a sample code to get the on-device LED to trigger on press and then from there I adapted things to wiring a button to stop the LED.

// 04.11.21 - Arduino + Machine Learning Exploration: TinyML using Tiny Motion Trainer and Arduino with Complex Gestures.

Now that the BLE connection works I think I can try to explore more complex gestures and then get them to trigger different outputs. I decided on a set of gestures that I do on a daily basis while working on coding related projects. I call them the working phase (normal state), idle/waiting phase (anxious state) cause I'm unsure if the code works, slam table (angry state) when it doesn't work, and happy shaking (excited state) when it eventually does work. I won't re-share the same kind of info like the screenshots and excel sheet from the previous Tiny Motion Trainer test as they're just data that I have not had the opportunity to find a nice way to package, but I'll focus on sharing the motions that I'm capturing this time.

This first set of gestures captured didn't work too well as some of the gestures had very similar movements to each other and that cause for the model to mix them up and give inaccurate results. So I decided to maintain the same idea but just tweak some of the movements so they're distinctly different from each other. I also realised that sometimes if the motion is too subtle even with a lower threshold, It won't detect it.

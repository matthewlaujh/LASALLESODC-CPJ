// 27.08.21 — Arduino Exploration: Explorations in MQTT Server Communication.

After some reading up on MQTT-protocols, Mainly the Arduino library page on communication protocols I checked out this link uno-wifi-rev2/uno-wifi-r2-mqtt-device-to-device to test to see if MQTT device to device communication would work.

Image: Serial Monitor showing messages being sent by the sender Arduino.

Image: Serial Monitor showing messages being receieved by the reciever Arduino.

With the MQTT-communication working, I began to try to implement it into my previously frankenstein-ed code and tried to understand further how it works. Through trial and error I thought I got quite close to getting it to work, cause the sender was sending the commands but the receiver wasn't receviing it, back to discord for some advice.

Discord - Mindaugas

where you set "topic", https://github.com/arduino-libraries/ArduinoMqttClient/blob/master/examples/WiFiSimpleReceive/WiFiSimpleReceive.ino you forgot to subscribe for specific topic

Discord - Mindaugas

I think problem is here because you drain available data and then trying to read something on empty buffer assign mqtt read to command and then print command also example from github doesnt have mqtt.poll in loop on receiver.

while (mqttClient.available()) {
Serial.print((char)mqttClient.read());
char command = mqttClient.read();"

Discord - AnonEngineering

What if you leave the "while" and take out the "Serial.print((char)mqttClient.read());"? each read removes a char from the buffer, with two you empty the buffer early like Mindaugas said.

Ran into some issues using the MQTT getting the devices to talk to each other and sought help from the discord once again. Turns out I forgot to set the Arduinos to subscribe to the specific topic, once I fixed that it worked, however having some issues with the reciever not reciving the correct input commands consistently and learnt that it was a debounce issue as well as one of the functions was emptying the buffer too early hence causing the inconsistency issues. But once those were resolved it worked perfectly! I even tried connecting on Arduino to the school's WIFI and the other to my phone's personal hotspot and it worked! I also learned about the secret tab in the Arduino IDE for sensitive information such as passwords which was quite informative and could come in handy next time.
